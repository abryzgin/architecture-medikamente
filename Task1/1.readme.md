## 3. Рекомендации по улучшению

| Категория                      | Предлагаемые меры                                                                                                                                                                                                  |
| ------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Классификация и защита данных** | • *Конфиденциальные данные*(PII, мед. тайна)→ **сквозное шифрование** (в покое и при передаче)<br>• *Внутренние данные* → **обфускация** (маскирование, хеширование)<br>• *Публичные данные* → **обезличивание** |
| **Контроль доступа**           | • Внедрение **RBAC / ABAC**<br>• Централизованная аутентификация через **Active Directory**<br>• **2FA** для критических систем                                                                                    |
| **Система хранения**           | • Миграция Excel → защищённая БД (PostgreSQL с TDE)<br>• Внедрение единой **CRM**<br>• Интеграция лабораторных API через **защищённый шлюз**                                                                       |
| **Мониторинг и аудит**         | • Сквозное **логирование действий** пользователей<br>• Интеграция в **SIEM** (ELK‑стек)<br>• Алёртинг при аномалиях (массовые выгрузки)                                                                            |
| **Тегирование**         | • Авто‑тегирование («PII», «Medical‑Secret» и т. д.)<br>• **Apache Atlas** для Data Lineage                                                                                                                        |
| **Резервирование и DLP**       | • Ежедневные **шифрованные бэкапы**<br>• Внедрение **DLP‑системы**                                                                                                                                                 |
| **Обучение персонала**         | • Регулярные тренинги по 152‑ФЗ и GDPR‑подходам<br>• Периодическое тестирование знаний                                                                                                                             |

## Итог

1. **Выявлены** все ключевые потоки конфиденциальных данных и задокументированы в DFD.
2. **Аудит** показал критические зоны: отсутствие сквозного шифрования, разрозненный контроль доступа, слабый мониторинг.
3. **Предложены** конкретные меры (шифрование, RBAC/ABAC, DLP, SIEM, Apache Atlas) и дополнительно обучающий контур для персонала.

Эти шаги формируют основу для перехода к решению отвечающему современным требованиям к приватности и безопасности данных.
